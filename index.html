Algorithms & Complexity

Asymptotic Analysis

Big O notation describes the execution time or space used by an algorithm relative to the input size.

// Time Complexity: O(log n) Int binarySearch(int arr[], int l, int r, int x) { While (l <= r) { Int m = l + (r – l) / 2; If (arr[m] == x) return m; If (arr[m] < x) l = m + 1; Else r = m – 1; } Return -1; }

Advanced Data Structures

  • Red-Black Trees: Self-balancing binary search trees ensuring O(log n) operations.
  • Graph Adjacency Lists: Space-efficient network representations using linked arrays.
  • Min/Max Heaps: Complete binary trees optimizing priority queue lookups.

Database Engineering

ACID Transactions

Relational storage compliance guarantees system data safety during hardware failures.

Property Definition Failure Impact
Atomicity All operations succeed or all fail entirely. Prevents partial saves.
Consistency Database moves only between valid database states. Prevents corrupt schemas.
Isolation Concurrent transactions run without interference. Prevents race conditions.
Durability Committed changes survive system crashes. Prevents lost mutations.

Networking & Protocols

The ISO-OSI Layer Model

Standard abstraction frames operational functions across distinct logical transport tiers.

  1. Application Layer: HTTP, FTP, DNS interfaces.
  2. Presentation Layer: Data encryption and format parsing.
  3. Session Layer: Inter-host socket management loops.
  4. Transport Layer: End-to-end TCP packet stream validation.
  5. Network Layer: IP address routing optimizations.
  6. Data Link Layer: MAC frames routing across physical mediums.
  7. Physical Layer: Raw bits over copper, glass, or air.
